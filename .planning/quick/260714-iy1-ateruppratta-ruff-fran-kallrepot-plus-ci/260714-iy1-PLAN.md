# Quick 260714-iy1 — Återupprätta ruff (från källrepot) + CI som faktiskt kör testerna

## Objective

**Detta är ingen ny konvention — det är en regression som rättas.** Källrepot
(`ai-agent-platform/services/agent/pyproject.toml`) har ruff. Price-trackers `pyproject.toml`
skrevs nytt i Fas 1 (Plan 01-01) och tappade den. Det är *därför* planeringsdokumenten hela tiden
antagit att ruff finns — den fanns, i originalet.

Dessutom: **repot har ingen CI som kör testerna.** `release.yml` är den enda workflowen och bygger
bara en image på tagg. De 214 gröna testerna är gröna enbart för att någon råkat köra dem lokalt.
En linter utan CI är en linter man glömmer.

## Uppmätt utgångsläge (kört, inte gissat)

Med källans exakta regeluppsättning, `ruff 0.14.4`, `--target-version py312 --line-length 100`:

```
32  E501   line-too-long        <- formattern löser dessa
 7  I001   unsorted-imports     [*] auto-fix
 3  F401   unused-import        [*] auto-fix
 2  UP017  datetime-timezone-utc[*] auto-fix
 1  UP032  f-string             [*] auto-fix
 1  UP041  timeout-error-alias  [*] auto-fix
--- 46 totalt, 14 auto-fixbara, NOLL säkerhetsfynd (S/bandit) ---
ruff format: 16 av 44 filer, 306 rader
```

## Task 1 — ärv ruff-konfigen från källrepot

**files:** `pyproject.toml`, `poetry.lock`

**action:**

Lägg till i dev-dependencies: `ruff = "0.14.4"` (samma pin som källan — konsistens över repogränsen).

Lägg in konfigen, **ärvd från källan** med tre nödvändiga anpassningar:

```toml
[tool.ruff]
line-length = 100
target-version = "py312"          # källan: py311 — vi kör 3.12
src = ["src"]

[tool.ruff.lint]
select = ["E", "F", "I", "B", "UP", "S", "N"]
ignore = ["S101", "S104", "B008", "PLR0912", "PLR0915"]

[tool.ruff.lint.isort]
known-first-party = ["domain", "api", "infra", "mcp_server"]   # källan hade sina egna paket

[tool.ruff.lint.mccabe]
max-complexity = 18

[tool.ruff.lint.per-file-ignores]
"tests/**/*.py" = ["S105", "S106"]   # hårdkodade "lösenord" i fixtures — som källan
```

**`B008` MÅSTE vara ignorerad.** Det är FastAPI:s `Depends()`-mönster i default-argument. Källan
ignorerar den av exakt samma skäl. **"Fixa" den aldrig** — att flytta ut `Depends()` ur
default-argumentet bryter hela dependency-injectionen. Detsamma gäller `S101` (assert i tester).

**verify:**
- `poetry run ruff --version` → `0.14.4`
- `poetry check` går igenom
- Konfigen läses: `poetry run ruff check --show-settings src | head` visar line-length 100

**done:** ruff är en deklarerad dev-dep med källans regler.

## Task 2 — kör format + auto-fix, städa resten

**files:** `src/**/*.py`, `tests/**/*.py`

**action:**

1. `poetry run ruff format src tests` (16 filer, ~306 rader). `ruff format` är black-kompatibel och
   **ersätter black** — dra inte in black separat.
2. `poetry run ruff check --fix src tests` (14 auto-fixbara).
3. Kör om `ruff check`. Det som återstår ska antingen åtgärdas för hand **eller** medvetet ignoreras
   med en motiverad `# noqa: XXX` — aldrig genom att slappa i konfigen.

**KRITISKT — sviten är sanningen, inte lintern.** Efter varje steg: `poetry run pytest -q`.
Om en fix gör ett test rött, **backa fixen** — den var fel. Auto-fix på `datetime-timezone-utc`
(UP017) och `f-string` (UP032) rör riktig logik, inte bara stil.

Rör **inte** `alembic/versions/0001_initial.py`s semantik. Formatering är okej; innehåll är det inte.

**verify:**
- `poetry run ruff check src tests` → `All checks passed!`
- `poetry run ruff format --check src tests` → inga filer att formatera
- `poetry run pytest -q` → **214 passed** (samma som före; inte 213, inte 215)

**done:** noll lint-fynd, sviten oförändrat grön.

## Task 3 — CI som faktiskt kör testerna

**files:** `.github/workflows/ci.yml` (NY)

**action:**

Repot har i dag **ingen** test-CI. Skapa `.github/workflows/ci.yml`:

- Triggar på `push` och `pull_request` (inte bara taggar — det är `release.yml`:s jobb).
- Python 3.12, poetry, `poetry install`.
- Steg: `ruff check src tests` → `ruff format --check src tests` → `pytest -q`.
- **Postgres-tjänst** så att DB-integrationstesterna faktiskt körs i CI i stället för att skippas.
  Om det blir krångligt: låt dem skippa (sviten är grön ändå, `202 passed, 12 skipped`) och
  **notera det i SUMMARY** — skippade tester i CI är en lucka, inte en seger.
- Rör **inte** `release.yml`.

**verify:**
- `.github/workflows/ci.yml` finns och är giltig YAML (`python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/ci.yml'))"`)
- Workflowen kör `pytest` — bekräfta med grep, inte med tro

**done:** en push kör tester + lint automatiskt.
