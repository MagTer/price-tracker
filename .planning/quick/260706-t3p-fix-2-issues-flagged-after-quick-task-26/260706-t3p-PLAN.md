Change `get_mcp_app()` to return a 2-tuple `(app_to_mount, http_app)` in both branches: when `MCP_BEARER_TOKEN` is set, the first element is the existing `BearerTokenMiddleware(http_app, MCP_BEARER_TOKEN)` wrapper (unchanged wrapping behavior); when it is not set, the first element is the raw `http_app` itself (same unprotected-with-a-warning behavior as today). In both branches the second tuple element is the same raw `http_app` local variable, so the caller can always reach `.lifespan` regardless of whether bearer wrapping was applied.

Update the function's docstring to state that it now returns a `(wrapped_app, http_app)` tuple and explain why the raw app is also returned: `app.mount()` in Starlette does not propagate a mounted sub-app's own lifespan, so the raw app must be exposed separately for the caller to enter `http_app.lifespan(http_app)` itself alongside its own startup/shutdown logic.
