# Quick Task 260714-x48: Flödesanalysens elva åtgärder + LLM-berikning Summary

**One-liner:** LLM enrichment at two precise triggers (first success / effective-price
drop, JSON-LD source only) plus the ten other flow-analysis fixes: single
`perform_price_check` flow, scheduler +1h/+24h retry semantics, weekly summary lowest
kr/enhet without the restart guard, 7-day latest-per-link deals, batched MCP store
names, truthful check_price docstring, kr/enhet-only create-watch form, Swedish email
diacritics, JSON-LD carousel-name sanity, and an OPENROUTER_API_KEY startup warning —
released as 0.4.0 (tag cut by orchestrator after merge).

## What was done (per finding)

1. **LLM enrichment (item 1).** `PriceParser.enrich_with_llm` reuses the already-fetched
   page, runs the (now single) `_run_llm_cascade`, and merges ONLY fields the JSON-LD
   base lacks — `price_sek`/`in_stock`/`confidence` always stay the base's. Never raises;
   below-floor or failed cascades return base unchanged. `raw_response` keeps
   `source == "jsonld"` and gains `enriched: True` + `enrichment_confidence`, so the
   scheduler's `checks_jsonld` stat stays truthful.
2. **Retry loop (item 2).** An exception-throwing link is backed off `next_check_at + 1h`
   via its own short session (own try/except; a dead DB cannot kill the loop). A failed
   (non-exception) check on a weekday link reschedules +24h instead of a full week.
3. **Weekly summary (item 3).** Restart guard deleted (in-memory `_last_summary_date`
   dedup remains — rare Monday-afternoon-restart duplicate accepted, locked). "Lägsta
   pris" is now the lowest kr/enhet over the latest point per link (effective price,
   `domain.pricing.unit_price_py`), with a lowest-absolute-price fallback labeled plain
   "kr"; the notifier renders the per-row label.
4. **Deals window (item 4).** `get_current_deals` selects the LATEST point per link
   within 7 days — a superseded offer can never appear; the redundant seen-set dedupe
   removed. Recency ordering deliberately kept.
5. **MCP list_products (item 5).** New `service.get_store_names_by_product` (one query,
   deduped + sorted names per product); "Ej länkad" only for genuinely unlinked products.
   Dead `get_stores()`/`store_map` plumbing removed from the tool.
6. **Consolidation (item 6).** `PriceTrackerService.check_price` DELETED. Module-level
   `service.perform_price_check` is the single flow; scheduler `_check_single_product`
   is a thin stats/alerts wrapper returning the outcome, and admin `trigger_price_check`
   delegates and commits, byte-compatible on both JSON shapes (400/404/502/500 contract
   pinned by new tests). Prior-point query keyed on `product_store_id` — AST gate green.
7. **check_price docstring (item 7).** Now states it returns the last OBSERVED price
   from stored history and does NOT trigger a live fetch.
8. **Watch form (item 8).** "Målpris (kr)" input removed from the create modal;
   `handleCreateWatch` sends `target_price_sek: null` unconditionally. Watch-list display
   of existing targets and the kr/enhet input untouched. JS fragment verified via the
   awk + `test -s` + `node --check` incantation (Gotcha 2); modal HTML spot-checked intact.
9. **Email diacritics (item 9).** målpris, Jämförelsepris(-fall), jämförelsepris,
   "Veckans prisöversikt – Prisspaning" (en dash), Här är, Du får/för, Lägsta pris.
10. **Startup warning (item 10).** `lifespan` logs an ERROR (English, developer-facing)
    when `OPENROUTER_API_KEY` is missing — no crash, mirrors the MCP_BEARER_TOKEN pattern.
11. **JSON-LD name sanity (item 11).** Zero overlap of 3+-char alphanumeric tokens
    (åäö included) between tracked name and node name → warn + return None so the cascade
    falls to the LLM; `product_name=None`, empty node names, and any overlap unchanged.
