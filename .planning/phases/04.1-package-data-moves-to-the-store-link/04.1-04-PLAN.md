<objective>
Close the trapdoor D-01 opens. Dropping `uq_product_store(product_id, store_id)` makes "two pack sizes at
one store" legal in the schema — and immediately fatal in three code paths that resolve a link by that pair
with `.scalar_one_or_none()`. Those queries are single-valued ONLY because of the constraint being dropped.
The first time the operator adds "Lambi 8-pack at Willys" beside the 24-pack — the phase's own acceptance
scenario — they raise `MultipleResultsFound` and surface as HTTP 500.

Purpose: MODEL-02 is not merely a DDL change. Re-keying these endpoints IS part of it.

Output: link endpoints re-keyed to `/product-stores/{product_store_id}`; the bulk-import path deduping on
`store_url`; package fields moved from the product schemas to the link schemas with their `> 0` validation
intact; duplicate URLs returning 409; and a static gate that fails the build if a pair-keyed lookup ever
comes back.
</objective>
