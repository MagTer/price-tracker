<objective>
Make "cheapest per unit" answerable over MCP. `compare_stores` currently emits one row per STORE, deduping
on store name — which erases exactly the distinction this phase creates (two pack sizes at one store) and
picks an arbitrary winner. D-13 makes it one row per LINK, sorted by kr/unit.

Purpose: the `priser` skill is the platform's main consumer of this repo. If the agent cannot see per-link
unit prices, the phase's core value never reaches the user who asked the question.

**This is also a bug fix, and the operator should know.** `compare_stores` reads `unit_price_sek` and
`in_stock` off `get_price_history()` rows — and neither key has ever been in that dict. The Jämförspris
column has printed `N/A` for every row since extraction, and the Lager column has printed `Nej` for every
row regardless of stock. The `priser` skill has never actually seen a comparison price.

Output: a per-link, kr/unit-ranked `compare_stores`; `check_price`'s stock flag fixed for free; updated
MCP tests.
</objective>
